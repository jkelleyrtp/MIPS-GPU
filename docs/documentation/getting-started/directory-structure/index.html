Here's a high level overview of each of these folders and config.toml.

ðŸ”—config.toml

A mandatory configuration file of Zola in TOML format. It is explained in details in the Configuration page.

ðŸ”—content

Where all your markup content lies: this will be mostly comprised of .md files. Each folder in the content directory represents a section that contains pages : your .md files.

To learn more, read the content overview.

ðŸ”—sass

Contains the Sass files to be compiled. Non-Sass files will be ignored. The directory structure of the sass folder will be preserved when copying over the compiled files: a file at sass/something/site.scss will be compiled to public/something/site.css.

ðŸ”—static

Contains any kind of files. All the files/folders in the static folder will be copied as-is in the output directory. If your static files are large you can configure Zola to hard link them instead of copying by setting hard_link_static = true in the config file.

ðŸ”—templates

Contains all the Tera templates that will be used to render this site. Have a look at the Templates to learn more about default templates and available variables.

ðŸ”—themes

Contains themes that can be used for that site. If you are not planning to use themes, leave this folder empty. If you want to learn about themes, head to the themes documentation.
